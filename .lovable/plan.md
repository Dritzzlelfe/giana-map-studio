# Giana Allen Design вҖ” Project Map

A persistent, editable XMind-style mind map backed by Supabase. Every edit autosaves; reload restores the full tree exactly.

## 1. Backend (Lovable Cloud / Supabase)

Enable Lovable Cloud, then ship a single migration that creates schema **and** seeds data server-side (so the seed runs exactly once, not from the client).

### Tables
- `maps(id uuid pk default gen_random_uuid(), name text not null unique, created_at timestamptz default now())`
  - `name` is `UNIQUE` so an accidental re-seed can't create a duplicate map.
- `map_nodes(id, map_id fkвҶ’maps on delete cascade, parent_id fkвҶ’map_nodes self on delete cascade, title, description, category, status, priority, color, sort_order numeric not null default 0, collapsed bool default false, created_at, updated_at)`
  - `sort_order` is **`numeric`** (not integer) so inserting between two siblings uses the midpoint of neighbors вҖ” no renumbering, no broken order. Reconciles the earlier inconsistency.
- Indexes on `map_nodes(parent_id)` and `map_nodes(map_id)`.
- `updated_at` trigger.

### Seed
Done **inside the migration** with a recursive `INSERT` building the full hierarchy from the spec (priority `asap` for any line containing `[asap]`). No client-side "if empty then insert" вҖ” that pattern double-fires under React StrictMode and races across tabs. Idempotent guard: `INSERT ... WHERE NOT EXISTS (SELECT 1 FROM maps WHERE name = 'вҖҰ')`.

### RLS вҖ” explicit security posture for v1
- RLS **enabled** on both tables.
- v1 policies: `anon` + `authenticated` full CRUD. **This means the database is effectively public** to anyone with the project URL + publishable key. That is acceptable for a throwaway functional map, **not** for real client/financial data.
- Hard rule recorded in security memory + README: **no real client names, vendor prices, GAD margins, supplier terms, or invoicing data in this instance until auth + `auth.uid()`-scoped policies are in place.** This debt is settled *before* the data phase, not after.
- Schema is auth-ready: when auth is added, policies tighten to `owner_id = auth.uid()` (column added then) with no structural migration.
- Grants: `SELECT, INSERT, UPDATE, DELETE` to `anon` and `authenticated`; `ALL` to `service_role`.

## 2. Data layer

- `src/integrations/supabase/client.ts` (auto-generated by Cloud).
- `src/lib/mapStore.ts`: `loadMap`, `createNode`, `updateNode`, `deleteNode`, `reorderNode`, `reparentNode`.
- TanStack Query + optimistic mutations; rollback + toast on error.
- **Realtime: off for v1.** Optimistic writes + realtime echo cause flicker/duplicates unless self-events are deduped; not worth the complexity now. Revisit when multi-user lands.

### Mind Map view
- React Flow + `dagre` for left-to-right tree layout.
- Custom node: rounded card, title, category color dot, `ASAP` badge when priority=asap. Root larger, distinct color.
- Expand/collapse per node (persists `collapsed`).
- Hover/select toolbar: add child, add sibling, edit, delete.
- Pan/zoom, minimap, fit-to-view.

### Outline view
Indented collapsible list off the same tree. Inline rename. Reliable fallback always available via toggle.

### Detail drawer (shadcn Sheet)
Edit title, description, category, status, priority, color. Saves on blur/change.

### Delete confirmation
AlertDialog shows the **count of descendants that will be cascade-deleted** ("This will permanently delete *Schedules by Trade* and its **63 descendants**."). Computed from the in-memory tree before opening the dialog.

### Other
- Search filters + highlights matching nodes in both views.
- Export: JSON (full tree) and Markdown indented outline.
- Category legend in the header.

## 4. Visual style

Light, professional, calm. Rounded-xl cards, soft shadows, generous spacing. Serif display for the app title + sans for body. Category palette as semantic CSS tokens in `src/styles.css` (root, rooms, schedule, trade, furniture, kitchen, logistics, budget, coordination, data_model, roster, roadmap, deferred, field, action).

## 5. Build order

1. Enable Cloud вҶ’ migration with schema **and** seed вҶ’ load round-trip in Outline view (proves persistence).
2. Mind Map view with dagre + custom node.
3. Detail drawer + add/edit/delete with descendant-count confirmation.
4. Search, export, collapse persistence.
5. Polish; drag-to-reparent if time allows.

## Technical notes

- Browser Supabase client only (publishable key); no server functions for v1.
- Tree rebuilt in memory from flat rows on load; React Flow nodes/edges via `useMemo`.
- `sort_order` midpoint insert: new = `(prev + next) / 2`; first = `next - 1`; last = `prev + 1`. With `numeric`, no rebalancing needed for realistic depths.
- Drag-to-reparent updates `parent_id` + `sort_order` in one mutation.
- Export Markdown = recursive `  `-indented `- title`.
- Security memory updated on first migration to record the "no real data until auth" rule.
